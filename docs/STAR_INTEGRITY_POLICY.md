# Star Integrity Policy

OpenCLAW-P2P treats GitHub stars as a public trust signal. The project should grow through documentation quality, reproducible demos, research credibility, useful integrations, and legitimate community interest.

## Policy

We do not buy stars, ask for artificial stars, use bots, coordinate fake engagement, or run mass outreach campaigns.

Acceptable growth work:

- improve README clarity and onboarding
- publish reproducible demos and examples
- create citation metadata and release notes
- submit guideline-compliant PRs only where maintainers explicitly accept that category of contribution
- respond politely to direct maintainer questions
- stop immediately when maintainers decline or ask not to be contacted

Unacceptable growth work:

- paid or automated GitHub stars
- fake accounts or engagement rings
- repeated comments or bumps
- unrelated promotional issues
- cross-posting the same pitch into unrelated repositories
- re-opening declined submissions without invitation

## If suspicious star activity appears

If unusual star activity is observed, the project should:

1. Review the public star timeline and account-quality signals.
2. Check whether a real external source explains the traffic.
3. Avoid amplifying unsupported accusations.
4. Contact GitHub Support if there is credible evidence of artificial activity affecting the repository.
5. Document any confirmed remediation transparently.

## Community stance

The project is trying to earn attention by becoming easier to inspect, reproduce, cite, and integrate. Visibility work should leave durable technical value even if it produces no immediate stars.
