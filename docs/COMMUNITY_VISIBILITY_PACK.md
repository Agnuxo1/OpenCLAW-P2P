# Community Visibility Pack

This pack provides concise, reusable copy for describing OpenCLAW-P2P without mass outreach or promotional noise. Use it for GitHub profiles, release notes, personal posts, conference forms, project directories, and invited submissions.

## One-line description

OpenCLAW-P2P is a decentralized, agent-native research network for publishing, validating, and preserving scientific claims with Lean 4 verification, MCP tooling, and peer-to-peer infrastructure.

## Short description

OpenCLAW-P2P connects AI agents and human researchers in a peer-to-peer research network. It combines formal verification, content-addressed persistence, and agent-compatible interfaces so scientific claims can be published, checked, cited, and preserved with stronger provenance.

## Submission checklist for external directories

Before submitting OpenCLAW-P2P or a satellite project to an external list, verify:

- The target repository explicitly accepts project submissions.
- The project fits the listed category.
- The target repository has not previously declined further contact.
- The submission follows the repository's exact template.
- The text is factual and concise.
- The link points to the most relevant repository, usually OpenCLAW-P2P.
- No duplicate issue or PR already exists.

If a maintainer closes the submission as not planned, off-topic, or unwanted, do not reopen or re-submit unless they explicitly invite it.

## Weekly visibility routine

1. Review GitHub notifications and email once.
2. Reply only to direct questions or actionable maintainer feedback.
3. Label closed or declined threads as no follow-up.
4. Improve one owned repository artifact: README, docs, release notes, examples, screenshots, or citations.
5. Submit at most a small number of external PRs, only where guidelines clearly allow them.
6. Record blocked/no-contact repositories internally.
