# Contributing to OpenCLAW-P2P

OpenCLAW-P2P welcomes focused contributions that improve correctness, reproducibility, documentation, formal verification, agent integration, or the live research workflow.

## Good first contributions

- Fix unclear documentation or broken links.
- Add reproducible examples for agents, researchers, or MCP clients.
- Improve Lean 4 proof organization or documentation.
- Add tests, validation scripts, or small verification examples.
- Improve setup instructions for the live app, gateway, or local integrations.
- Report bugs with enough detail to reproduce them.

## Before opening a PR

1. Check existing issues and pull requests.
2. Keep the change scoped to one topic.
3. Explain why the change matters.
4. Include reproduction steps or validation output when applicable.
5. Do not include secrets, API keys, private data, or generated bulk content.

## Pull request checklist

- [ ] The change has a clear purpose.
- [ ] Documentation is updated when behavior changes.
- [ ] Commands, tests, or manual validation steps are included in the PR description.
- [ ] The PR does not add unrelated promotional material.
- [ ] The PR follows the repository's non-spam visibility policy.

## Community rules

We do not use mass comments, repeated unsolicited outreach, or unrelated promotional issues. External visibility work should happen through useful documentation, reproducible demos, invited discussions, and PRs that match another project's published contribution guidelines.

## Security and private reports

Please do not open public issues for secrets, vulnerabilities, or private operational details. Use the process in `SECURITY.md` instead.
